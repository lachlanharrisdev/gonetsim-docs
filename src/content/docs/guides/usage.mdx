This document will guide you through GoNetSim's basic commands.

## Prerequisites

* GoNetSim installed **standalone** on your system. If you haven't done this yet, please see the [installation guide](/guides/installation). This tutorial does not directly walk through dockerised setups.

<Aside>This tutorial assumes you have a basic understanding of the command line and are comfortable navigating your file system.</Aside>

## Baby's First GoNetSim

To start GoNetSim, you can simply run the `gonetsim` command (or `gonetsim.exe` on windows)

<Tabs>
	<TabItem label="Linux">
		```bash
		gonetsim
		```
	</TabItem>
	<TabItem label="Windows">
		```bash
		gonetsim.exe
		```
	</TabItem>
</Tabs>

The output will show what services have started & on what ports.

### Configuration

After running `gonetsim` for the first time, a default config file will be generated at `~/.config/gonetsim/config.toml` (or `%APPDATA%\gonetsim\config.toml` on Windows). Configuration files are the primary way to enable, disable & modify services in GoNetSim.

Let's try disabling the DNS service:

<Steps>
1. Open the config file at `~/.config/gonetsim/config.toml` (or `%APPDATA%\gonetsim\config.toml` on Windows) in a text editor.

2. Find the `[services.dns]` section and set `enabled = false`.

    ```diff
    ...
    [dns]
    - enabled = true
    + enabled = false
    listen = ":5354"
    network = "udp"
    ipv4 = "127.0.0.1"
    ipv6 = "::1"
    ...
    ```


3. Save the file

4. Run `gonetsim` (or `gonetsim.exe` on Windows) again to confirm the changes.

    <Tabs>
    <TabItem label="Linux">
    ```bash
    gonetsim
    ```
    </TabItem>
    <TabItem label="Windows">
    ```bash
    gonetsim.exe
    ```
    </TabItem>
    </Tabs>
</Steps>

If you'd like to learn more about configuration, please see the [configuration guide](/guides/configuration).
