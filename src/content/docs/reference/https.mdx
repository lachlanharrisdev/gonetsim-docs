The HTTPS service simulates a basic web server. It extends the functionality of the [HTTP service](/reference/http) by adding TLS support. It has two modes of operation:

<Tabs>
<TabItem label="Fake">
In "Fake" mode, requests to any URL will return a generic response with a configurable status code, headers and body. 
</TabItem>
<TabItem label="Real">
In "Real" mode, a directory can be specified and the server will serve static files from that directory. If a requested file is not found, a 404 response will be returned.

<Aside type="caution">"Real" mode is still in development. Please see the [GitHub Issue](https://github.com/lachlanharrisdev/gonetsim/issues/13) for its current status of development</Aside>
</TabItem>
</Tabs>


## Configuration

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | bool | `true` | Whether the HTTPS service is enabled |
| `listen` | string | `:8443` | The address and port to listen on (e.g. `:443`) |
| `status` | int | `200` | The status code to return in "Fake" mode |
| `cert` | string | `""` | The path to the TLS certificate file (PEM format). If empty, a self-signed certificate will be generated |
| `key` | string | `""` | The path to the TLS key file (PEM format). If empty, a self-signed key will be generated |

To see instructions on how to configure these options, see the [configuration guide](/guides/configuration).

## Command

The HTTPS service can be run individually using the `https` subcommand:

<Tabs>
<TabItem label="Linux">
```bash
gonetsim https
```
</TabItem>
<TabItem label="Windows">
```bash
gonetsim.exe https
```
</TabItem>
</Tabs>

This starts the HTTPS service with the current configuration, without starting any other services (Including the HTTP service; the HTTP service is not started with this command). 

## Testing

You can test the HTTPS service using `curl`.

<Steps>
1. Start the HTTPS service

    ```bash
    gonetsim https
    ```
2. In a separate terminal, make a request to the service using `curl`
    <Aside>Since the service uses a self-signed certificate by default, you will need to use the `-k` option to allow insecure connections.</Aside>

    ```bash
    curl -k https://localhost
    ```

3. Return to the GoNetSim instance and you should see a log of the request in the output

</Steps>
