The HTTP service simulates a basic web server. It has two modes of operation:

<Tabs>
<TabItem label="Fake">
In "Fake" mode, requests to any URL will return a generic response with a configurable status code & headers. Supported file extensions (currently `.html`, `.htm`, `.asp`, `.txt`, `.md`, `.js`, `.css`)
will return a realistic body of that type, and unsupported file extensions will return the default html page.
</TabItem>
<TabItem label="Real">
In "Real" mode, a directory can be specified and the server will serve static files from that directory. If a requested file is not found, a 404 response will be returned.

<Aside type="caution">"Real" mode is still in development. Please see the [GitHub Issue](https://github.com/lachlanharrisdev/gonetsim/issues/13) for its current status of development</Aside>
</TabItem>
</Tabs>


## Configuration

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `enabled` | bool | `true` | Whether the HTTP service is enabled |
| `listen` | string | `:8080` | The address and port to listen on (e.g. `:8080`) |
| `status` | int | `200` | The status code to return in "Fake" mode |

To see instructions on how to configure these options, see the [configuration guide](/guides/configuration).

## Command

The HTTP service can be run individually using the `http` subcommand:

<Tabs>
<TabItem label="Linux">
```bash
gonetsim http
```
</TabItem>
<TabItem label="Windows">
```bash
gonetsim.exe http
```
</TabItem>
</Tabs>

This starts the HTTP service with the current configuration, without starting any other services (Including the HTTPS service; the HTTPS service is not started with this command). 

## Testing

You can test the HTTP service using `curl`.

<Steps>
1. Start the HTTP service

    ```bash
    gonetsim http
    ```
2. In a separate terminal, make a request to the service using `curl`

    ```bash
    curl https://localhost:8080
    ```

3. Return to the GoNetSim instance and you should see a log of the request in the output

</Steps>
